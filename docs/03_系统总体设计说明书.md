# 系统总体设计说明书

## 1. 总体架构

系统采用前后端分离架构。前端通过 Vue 3 构建单页应用，负责登录、任务操作、图表展示、漏洞详情、日志时间线和报告预览；后端通过 Spring Boot 提供 REST API，负责认证、任务管理、Agent 模拟执行、结果分析、报告生成和数据库访问；MySQL 存储用户、任务、漏洞、日志和报告数据。

架构文字图如下：

```text
浏览器 Vue3 前端
  -> Axios 调用 REST API
Spring Boot 后端
  -> Controller 接口层
  -> Service 业务层
  -> Agent 规则模拟层
  -> Mapper 数据访问层
MySQL 数据库
```

## 2. 模块划分

- 用户认证模块：登录、token 生成、接口拦截。
- 仪表盘模块：统计任务、漏洞、风险等级、趋势和状态分布。
- 测试任务模块：任务 CRUD、分页筛选、状态流转。
- Agent 模块：测试计划生成、模拟执行、结果分析、报告生成。
- 漏洞结果模块：漏洞列表、分页筛选、详情展示。
- 测试日志模块：日志查询、级别筛选、时间线展示。
- 报告中心模块：报告查询、Markdown 预览、HTML 导出、打印 PDF。
- 系统说明模块：展示项目背景、架构和课程设计亮点。

## 3. 前后端交互说明

前端通过 Axios 封装统一请求，登录成功后将 token 存入 localStorage，并在请求头中携带 `Authorization: Bearer <token>`。后端拦截 `/api/**` 业务接口，校验 token 后再进入 Controller。接口统一返回 `ApiResponse`，分页接口返回 `PageResult`。

## 4. Agent 流程说明

Agent 当前采用规则模拟版，分为四个核心组件：

- `TestPlanGenerator`：根据目标地址、测试类型和深度生成计划。
- `TestExecutor`：模拟执行测试并生成日志。
- `ResultAnalyzer`：根据测试类型生成漏洞结果和修复建议。
- `ReportGenerator`：生成 Markdown 测试报告。

## 5. 部署结构

本地部署时，后端运行在 `8080` 端口，前端运行在 `5173` 端口，MySQL 运行在本机。前端 Vite 代理将 `/api` 请求转发到后端。

## 双端架构补充设计

系统在同一套 Vue 前端中实现管理员端和普通用户端，通过登录返回的角色信息动态生成菜单和首页入口。ADMIN 登录后进入 `/admin/dashboard`，USER 登录后进入 `/user/dashboard`。后端仍采用 Spring Boot 单体服务，通过拦截器解析 token，将当前用户 ID 与角色保存到 `UserContext`，业务接口根据角色决定数据范围。

### 权限控制流程

1. 用户登录后获取 token 和 user 信息。
2. 前端保存 token、userId、username、nickname、role。
3. 请求接口时携带 Authorization Bearer token。
4. 后端拦截器解析 token，写入当前用户上下文。
5. 管理员接口校验 role=ADMIN。
6. 任务、漏洞、日志、报告接口根据 task.create_user_id 判断资源归属。

## 创新模块架构

新增模块继续遵循前后端分离与后端分层设计：

- Agent 决策链模块：`agent_decision` 表保存任务执行解释过程，任务详情页以可视化卡片展示。
- 风险评分模块：`RiskScoreService` 根据漏洞类型、等级和场景生成简化 CVSS-Lite 评分。
- 整改工单模块：`remediation_ticket` 表记录漏洞整改生命周期，支持复测模拟。
- 安全知识库模块：`security_knowledge` 表沉淀漏洞知识，漏洞详情可联动查看。
- 操作审计模块：`audit_log` 表记录平台关键操作，仅管理员可查看。
- 内置演示靶场：`DemoTargetController` 提供本地模拟接口，保证答辩演示稳定。

## 趣味靶场模块位置

趣味靶场位于前端业务层和后端挑战服务之间，复用现有登录鉴权、角色权限、安全知识库、审计日志和内置演示靶场接口。

```text
Vue 趣味靶场页面
  -> ChallengeController / AdminChallengeController
  -> challenge / challenge_submit / user_score / user_achievement
  -> SecurityKnowledge / AuditLog / DemoTarget
```

普通用户通过靶场练习安全知识并获得积分，管理员通过题目管理维护题库。该模块增强项目的安全教育属性和答辩趣味性。
