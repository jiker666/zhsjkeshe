# 软件测试报告

## 1. 测试目标

验证第二轮完善后的系统是否满足课程设计演示要求，包括登录、仪表盘、任务分页筛选、Agent 状态流转、漏洞详情、日志筛选、报告导出和系统说明页。

## 2. 测试环境

- 操作系统：macOS / Windows / Linux 均可
- JDK：17 或以上
- Maven：3.x
- Node.js：18 或以上
- MySQL：8.x
- 浏览器：Chrome / Edge

## 3. 测试方法

采用黑盒功能测试和构建验证相结合的方式。功能测试覆盖核心业务流程，构建验证使用 `mvn test` 和 `npm run build`。

## 5. 构建验证

- 后端执行 `mvn test`：通过。
- 前端执行 `npm run build`：通过。

## 6. 缺陷与修复说明

- 第一轮页面偏基础后台模板，第二轮已改为深色安全运营控制台风格。
- 第一轮列表无分页，第二轮已增加任务、漏洞、日志分页接口。
- 第一轮 Agent 未抽象，第二轮已拆分为计划生成、执行、分析和报告组件。
- 第一轮演示数据较少，第二轮已补充完整初始化数据。

## 7. 测试结论

系统功能闭环完整，构建验证通过，能够支撑课程设计答辩演示。
