# 答辩演示流程

## 1. 登录系统

操作：打开前端地址，输入 `admin / 123456` 登录。

讲解词：本系统采用登录入口和 token 鉴权保护业务接口，体现了基础用户认证和权限控制。

## 2. 首页安全态势

操作：进入首页仪表盘，展示任务统计、漏洞统计、风险等级分布、任务趋势和高危漏洞列表。

讲解词：首页用于安全测试态势总览，可以快速掌握测试任务完成情况和风险分布。

## 3. 新建任务

操作：进入“新建任务”，点击“示例任务”自动填充，提交后进入任务详情。

讲解词：任务表单体现安全测试任务建模，包括目标地址、测试类型、测试深度、是否生成报告和任务描述。

## 4. Agent 执行

操作：在任务详情页点击“执行测试”。

讲解词：系统模拟 Agent 的任务理解、计划生成、自动执行、结果分析和报告生成过程，并通过状态流转展示。

## 5. 查看漏洞

操作：进入漏洞结果页，筛选风险等级，打开漏洞详情弹窗。

讲解词：漏洞详情包含风险等级、漏洞类型、描述、复现步骤、请求示例、响应示例和修复建议，便于安全测试结果复盘。

## 6. 修改漏洞处理状态

操作：在漏洞详情中将状态从“未处理”改为“已确认”或“已修复”。

讲解词：漏洞状态流转体现结果管理闭环，方便后续扩展修复跟踪流程。

## 7. 查看日志

操作：进入测试日志页，按任务或日志级别筛选。

讲解词：日志时间线展示 Agent 执行过程，便于追踪测试计划、执行和分析步骤。

## 8. 导出报告

操作：进入报告中心，选择任务，预览报告，导出 HTML、Word，或使用浏览器打印 PDF。

讲解词：报告中心将测试过程和漏洞结果整理为正式报告，是课程设计最终交付的重要展示点。

## 9. 系统说明

操作：打开系统说明页。

讲解词：系统说明页总结项目背景、设计目标、技术架构、Agent 流程、创新点和后续扩展方向，适合答辩收尾。

## 双端演示补充流程

1. 管理员端演示：使用 admin/123456 登录，展示管理员首页、用户管理、全部任务、全部漏洞和系统配置，强调管理员具备平台治理和全量审计能力。
2. 普通用户端演示：退出后使用 user/123456 登录，展示用户首页、我的任务、我的漏洞、我的日志、我的报告和个人中心，强调只能访问自己的数据。
3. 权限拦截演示：普通用户尝试访问 `/admin/users`，系统跳转无权限页或接口返回 403，说明前后端均做了权限控制。
4. 注册演示：在登录页点击立即注册，注册新安全测试人员账号，注册成功后返回登录页。

## 加分项演示补充流程

1. Agent 决策链：执行任务后在任务详情页展示 6 个决策阶段，讲解“系统不仅给结果，还记录 Agent 的判断依据”。
2. 风险评分：在漏洞结果页打开详情，展示风险评分、影响分、可利用性分和风险向量，说明漏洞优先级排序。
3. 整改工单：从漏洞详情生成整改工单，进入整改工单页修改状态并一键复测，展示发现到关闭的闭环。
4. 安全知识库：打开知识库页面，展示常见漏洞类型的成因、影响和修复建议，说明知识沉淀价值。
5. 操作审计：管理员进入操作审计页，展示登录、执行任务、导出报告、工单流转等记录。
6. 内置演示靶场：新建任务时点击“使用内置演示靶场”，说明演示目标为本地模拟接口，安全可控。

## 趣味靶场演示步骤

1. 打开“趣味靶场”，展示题目卡片、难度、类型、积分和通过率。
2. 进入“神秘公开接口”题目详情，展示题面、模拟请求、提示和关联知识点。
3. 先提交错误答案，展示输入框抖动和友好提示。
4. 再提交正确答案 `flag{public_info_leak}`，展示彩带动画、挑战成功弹窗、积分和成就徽章。
5. 点击“让 Agent 分析思路”，展示 Agent 只给分析方向，不直接给答案。
6. 进入“我的靶场成绩”，展示积分、成就、雷达图和积分趋势。
7. 使用管理员打开“靶场题目管理”，展示新增、编辑、启用禁用和提交记录。
