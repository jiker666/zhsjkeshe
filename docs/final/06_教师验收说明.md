# 教师验收说明

## 1. 如何启动

推荐使用一键脚本：

```bash
chmod +x scripts/*.sh
./scripts/start_all.sh
```

如数据库账号不是 `root/root`，请先设置：

```bash
export DB_USER=root
export DB_PASSWORD=实际密码
```

## 2. 如何登录

前端启动后访问脚本输出的地址，默认账号为：

```text
admin / 123456
user / 123456
```

## 3. 如何演示

按以下路径验收：

1. 登录系统。
2. 查看首页仪表盘。
3. 新建任务并使用示例任务。
4. 执行 Agent 模拟测试。
5. 查看漏洞详情和处理状态。
6. 查看日志时间线。
7. 进入报告中心导出 Word/HTML 或打印 PDF。
8. 查看系统说明页。
9. 打开任务详情查看 Agent 决策链。
10. 从漏洞详情生成整改工单并复测。
11. 打开安全知识库和管理员操作审计。
12. 新建任务时使用内置演示靶场。

## 4. 已完成功能

- 用户登录和基础角色控制。
- 测试任务管理和分页筛选。
- Agent 规则模拟执行。
- 漏洞结果和处理状态管理。
- 日志时间线。
- 报告预览和导出。
- 首页可视化统计。
- 最终交付文档和启动脚本。
- 管理员端和普通用户端双端模式。
- 注册、用户管理和个人中心。
- Agent 决策链、风险评分、整改工单、安全知识库、操作审计和内置演示靶场。

## 5. 课程设计创新点

项目将 AI Agent 工作流引入安全测试管理场景，形成任务编排、计划生成、模拟执行、结果分析和报告生成的闭环，兼顾软件工程流程完整性和可演示性。

新增加分项进一步体现“可解释、可量化、可闭环、可沉淀、可审计、可稳定演示”的课程设计特点：Agent 决策链解释每一步判断，风险评分帮助排序修复，整改工单体现安全治理闭环，知识库沉淀经验，审计日志保障平台可追溯，内置演示靶场降低答辩环境依赖。
