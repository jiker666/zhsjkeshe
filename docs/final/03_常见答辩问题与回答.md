# 常见答辩问题与回答

## 1. 为什么选择这个题目？

Web 安全测试流程包含任务、执行、结果、日志和报告，适合体现完整软件工程流程，同时 AI Agent 是当前较新的软件系统设计方向。

## 2. 和普通漏洞扫描器有什么区别？

本项目重点不是真实扫描能力，而是安全测试任务的可视化编排、Agent 流程模拟、结果管理和报告生成。

## 3. AI Agent 体现在哪里？

体现在任务理解、测试计划生成、模拟执行、结果分析和报告生成的流程抽象上，后端也将 Agent 拆成多个可替换组件。

## 4. 为什么没有接真实大模型？

课程设计第一目标是可运行和可演示。真实大模型涉及密钥、费用、网络和安全边界，因此第一版使用规则模拟，保留扩展接口。

## 5. 如何保证安全合规？

系统默认仅用于本地靶场、课程演示环境或已授权目标，不执行破坏性扫描，也不做公网爆破。

## 6. 后续如何接入真实扫描器？

可以在 `SecurityAgentService` 下新增实现类，将 `TestExecutor` 替换为真实扫描器适配器，并保持结果入库格式不变。

## 7. 数据库如何设计？

核心表包括 user、test_task、test_result、test_log、test_report，分别对应用户、任务、漏洞结果、日志和报告。

## 8. 系统采用什么架构？

采用前后端分离架构，前端 Vue 3，后端 Spring Boot，数据库 MySQL，通过 REST API 通信。

## 9. 有哪些可视化内容？

首页包含风险等级饼图、最近 7 天任务趋势、测试类型分布、任务状态分布和高危漏洞 Top。

## 10. 报告如何生成？

后端根据任务、漏洞结果和日志生成 Markdown 报告，同时支持服务端导出 Word，前端支持 HTML 导出和浏览器打印 PDF。

## 11. 权限如何实现？

系统支持 ADMIN 和 USER 角色。管理员可以删除任务和结果，普通用户主要用于查看和创建任务。

## 12. 测试怎么做？

通过功能测试和构建验证完成，后端执行 `mvn test`，前端执行 `npm run build`，并验证登录、统计、分页、报告导出等接口。

## 13. 项目创新点是什么？

将 AI Agent 工作流与安全测试任务管理结合，通过可视化方式展示测试计划、状态流转、漏洞分析和报告生成。

## 14. 如果后端端口被占用怎么办？

启动脚本会检测 8080，如果占用则自动切换到 18080，并同步更新前端 API 地址。

## 15. 后续如何完善？

可以接入真实大模型、真实授权扫描器、RBAC 权限模型、报告模板管理、修复闭环和多用户协作。

## 双端权限相关问答补充

**Q16：为什么要区分管理员端和普通用户端？**  
A：课程设计要求体现用户、权限和完整业务系统能力。管理员端负责平台治理和全量审计，普通用户端负责个人安全测试任务，两者职责清晰。

**Q17：普通用户如何保证不能看到别人数据？**  
A：前端通过菜单和路由守卫隐藏管理员入口，后端在任务、漏洞、日志和报告接口中根据 token 解析出的 userId 判断数据归属，越权时返回 403。

**Q18：注册用户的密码是否明文保存？**  
A：不是。注册和管理员创建用户时后端使用 SHA-256 对密码做哈希保存，登录时进行哈希匹配。

**Q19：Agent 决策链有什么作用？**  
A：它记录 Agent 每个阶段的输入、输出、工具和判断原因，让测试过程可解释、可复盘、可审计。

**Q20：风险评分怎么计算？**  
A：系统采用课程设计简化 CVSS-Lite 模型，根据漏洞类型、风险等级、影响范围、可利用性和利用难度生成 0.0 到 10.0 分，用于排序和优先级判断。

**Q21：整改工单体现了什么？**  
A：它体现漏洞发现后的治理闭环，从发现、确认、修复、复测到关闭，不只是停留在扫描结果展示。

**Q22：安全知识库有什么价值？**  
A：知识库沉淀常见漏洞的成因、影响和修复建议，后续可以作为 Agent 生成修复建议和自学习的基础。

**Q23：操作审计为什么重要？**  
A：安全平台自身也需要可追溯，审计日志可以记录关键操作，便于责任定位和安全管理。

**Q24：内置演示靶场是否安全？**  
A：安全。它只返回本地模拟数据，用于课程设计演示，不会对公网目标发起扫描，也不包含破坏性攻击逻辑。

**Q25：为什么要做趣味靶场？**  
A：它让系统从“安全测试平台”扩展到“安全教育平台”，答辩时更直观，也能展示积分、成就、题库管理等完整业务能力。

**Q26：靶场是否涉及真实攻击？**  
A：不涉及。所有题目都基于本地模拟接口和演示数据，只用于课程设计和安全意识训练。

**Q27：答案为什么要哈希存储？**  
A：避免前端或数据库中直接暴露答案，体现基本安全设计意识。后端用 SHA-256 对提交答案进行比对。

**Q28：积分和成就有什么意义？**  
A：积分和成就提升系统趣味性，也让用户学习进度可视化，适合课程设计答辩展示。

**Q29：GitHub 开源时如何避免风险？**  
A：README、SECURITY.md 和题目说明都明确安全边界；项目不含真实攻击扫描逻辑，演示接口仅返回本地模拟数据。
