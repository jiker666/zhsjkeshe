# 软件需求规格说明书

## 1. 系统目标

实现一个自主安全测试智能体可视化平台。用户登录后可以创建安全测试任务，系统自动生成测试计划，规则模拟 Agent 执行测试流程，生成日志、漏洞结果、修复建议和测试报告，并通过仪表盘进行可视化展示。

## 2. 用户角色

- 管理员：登录系统、创建和管理任务、执行测试、查看漏洞、日志和报告。
- 普通用户：后续可扩展为仅管理本人任务和查看授权数据。

## 3. 功能需求

### 3.1 用户登录

用户输入用户名和密码，登录成功后获得 token，后续业务接口需携带 token 访问。

### 3.2 首页仪表盘

展示总任务数、运行中任务数、已完成任务数、漏洞数、高危漏洞数、今日测试次数，并展示风险等级分布、最近 7 天任务趋势、测试类型分布和任务状态分布。

### 3.3 测试任务管理

支持任务新增、查询、分页、筛选、详情、删除和执行。任务字段包括任务名称、目标地址、测试类型、测试深度、是否生成报告、任务描述、测试计划和状态。

### 3.4 Agent 模拟执行

Agent 按任务理解、计划生成、自动执行、结果分析、报告生成的流程运行。状态包括 PENDING、PLANNING、RUNNING、ANALYZING、COMPLETED、FAILED。

### 3.5 漏洞结果管理

展示漏洞名称、漏洞类型、风险等级、影响地址、描述、复现步骤、请求示例、响应示例和修复建议，支持按风险等级和漏洞类型筛选。

### 3.6 测试日志

以时间线形式展示 Agent 执行过程，支持按任务和日志级别筛选。日志级别包括 INFO、WARN、ERROR、SUCCESS。

### 3.7 报告中心

支持按任务预览 Markdown 报告，报告内容包括任务信息、测试目标、测试范围、测试方法、风险统计、漏洞详情、日志摘要、修复建议和总结结论，并支持 HTML 导出和浏览器打印 PDF。

### 3.8 系统说明

展示平台背景、功能模块、技术架构、Agent 工作流程、适用场景、课程设计亮点和后续扩展方向。

## 4. 用例描述

| 用例 | 参与者 | 前置条件 | 基本流程 | 结果 |
| --- | --- | --- | --- | --- |
| 登录系统 | 管理员 | 用户存在 | 输入账号密码并提交 | 进入仪表盘 |
| 新建任务 | 管理员 | 已登录 | 填写表单或一键示例，提交 | 生成测试计划 |
| 执行测试 | 管理员 | 任务存在 | 点击执行测试 | 生成日志、漏洞和报告 |
| 查看漏洞 | 管理员 | 已执行任务 | 进入漏洞结果页并筛选 | 查看漏洞详情 |
| 预览报告 | 管理员 | 已生成报告 | 选择任务预览 | 查看或导出报告 |

## 5. 非功能需求

- 易用性：页面清晰，支持示例任务，适合答辩演示。
- 可维护性：后端分层清晰，前端页面职责明确。
- 可扩展性：Agent 抽象为接口，后续可接入真实大模型或扫描器。
- 安全性：业务接口通过 token 拦截，测试功能限定课程演示和授权范围。
- 性能：分页接口避免一次性加载大量数据。

## 6. 业务流程

用户登录 -> 首页仪表盘 -> 新建测试任务 -> Agent 生成计划 -> 执行测试 -> 状态流转 -> 生成日志和漏洞 -> 查看漏洞详情 -> 预览报告 -> 导出报告。

## 双端与权限补充需求

### 用户角色

系统划分为管理员 ADMIN 和普通用户 USER 两类角色。管理员负责平台治理、用户管理、全量任务审计和演示数据维护；普通用户作为安全测试人员，只能管理自己创建的测试任务及对应漏洞、日志和报告。

### 新增功能需求

1. 注册登录：访客可注册普通用户账号，注册时填写用户名、密码、昵称、邮箱和手机号，默认角色为 USER。
2. 管理员端：提供管理员首页、用户管理、全部测试任务、全部漏洞结果、测试日志、报告中心、系统配置和系统说明。
3. 普通用户端：提供用户首页、我的测试任务、我的漏洞结果、我的测试日志、我的报告、个人中心和系统说明。
4. 用户管理：管理员可分页查询用户、新增用户、编辑用户、启用/禁用用户、重置密码和删除用户，但不能删除当前登录管理员自己。
5. 个人中心：所有登录用户可查看并修改昵称、邮箱、手机号和密码。
6. 权限控制：普通用户不能访问管理员接口，不能查看或操作他人任务、漏洞、日志和报告。

### 权限需求

系统前端通过路由守卫控制页面入口，后端通过 token 中的 userId 和 role 进行接口级校验。越权访问返回 403，提示“无权限访问该资源”。

## 创新加分项需求

1. Agent 决策链：系统应记录 Agent 在任务理解、计划生成、工具选择、测试执行、结果分析和报告生成阶段的决策依据、输入输出摘要、使用工具和置信度。
2. 风险智能评分：漏洞结果除 HIGH/MEDIUM/LOW 外，应提供 0.0 - 10.0 的风险评分，以及影响分、可利用性分、利用难度分和风险向量。
3. 整改工单：系统应支持从漏洞生成整改工单，跟踪 TODO、CONFIRMED、FIXING、RETESTING、CLOSED、IGNORED 等状态。
4. 安全知识库：系统应提供常见漏洞类型的成因、影响、修复建议和参考说明，普通用户可查看，管理员可维护。
5. 操作审计：系统应记录登录、注册、任务执行、报告导出、漏洞状态修改、工单流转和用户管理等关键操作。
6. 内置演示靶场：系统应提供本地模拟目标，支持课程演示时不依赖外部靶场完成任务流程。

## 趣味安全靶场需求

1. 题目列表：用户可查看题目卡片，包含难度、类型、积分、完成状态和通过率。
2. 题目详情：用户可查看题目背景、目标、提示、模拟请求、关联知识库和 Agent 分析思路。
3. 答案提交：用户输入答案后提交，系统校验答案哈希，答错给友好提示，答对展示成功动画。
4. 积分成就：首次答对题目获得积分，累计完成题目后解锁“初次破冰”“安全新手”“靶场达人”等成就。
5. 我的成绩：展示总积分、完成题目数、正确率、连续答对次数、最近记录、成就徽章、类型掌握雷达图和积分趋势。
6. 管理员题目管理：管理员可新增、编辑、启用/禁用、删除题目，并查看提交记录。
7. 联动需求：题目应关联安全知识库、Agent 分析和审计日志，所有场景仅使用本地模拟数据。
