# 详细设计说明书

## 1. 核心模块设计

### 1.1 认证模块

登录接口为 `POST /api/auth/login`。后端根据用户名、密码和用户状态校验身份，成功后生成内存 token。业务接口通过拦截器读取 `Authorization` 请求头并设置当前用户上下文。

### 1.2 任务模块

任务模块负责测试任务的创建、查询、分页、筛选、详情、删除和执行。创建任务时自动调用 Agent 生成测试计划。执行任务时按状态流转写入日志。

任务状态：

- `PENDING`：待执行
- `PLANNING`：计划生成中
- `RUNNING`：执行中
- `ANALYZING`：结果分析中
- `COMPLETED`：已完成
- `FAILED`：执行失败

### 1.3 Agent 模块

Agent 模块抽象为 `SecurityAgentService`，当前实现为 `RuleBasedSecurityAgentServiceImpl`。该实现组合 `TestPlanGenerator`、`TestExecutor`、`ResultAnalyzer` 和 `ReportGenerator`，便于后续替换为真实 AI Agent。

### 1.4 漏洞模块

漏洞结果包含漏洞名称、漏洞类型、风险等级、影响地址、描述、复现步骤、请求示例、响应示例和修复建议。前端通过弹窗展示漏洞详情。

### 1.5 报告模块

报告模块生成 Markdown 格式报告。报告内容覆盖任务信息、测试范围、测试方法、风险统计、漏洞详情、日志摘要和结论。

## 2. 接口设计

| 接口 | 方法 | 说明 |
| --- | --- | --- |
| `/api/auth/login` | POST | 用户登录 |
| `/api/dashboard/stats` | GET | 首页统计 |
| `/api/tasks` | GET | 任务列表，支持分页和筛选 |
| `/api/tasks` | POST | 新建任务 |
| `/api/tasks/{id}` | GET | 任务详情 |
| `/api/tasks/{id}` | PUT | 修改任务 |
| `/api/tasks/{id}` | DELETE | 删除任务 |
| `/api/tasks/{id}/execute` | POST | 执行测试 |
| `/api/results` | GET | 漏洞列表，支持分页和筛选 |
| `/api/results/{id}` | GET | 漏洞详情 |
| `/api/logs` | GET | 日志列表，支持任务和级别筛选 |
| `/api/reports/task/{taskId}` | GET | 查询任务报告 |

## 3. 类设计

- `ApiResponse`：统一接口响应。
- `PageResult`：统一分页响应。
- `TestTaskServiceImpl`：任务业务逻辑和状态流转。
- `RuleBasedSecurityAgentServiceImpl`：规则模拟 Agent 总入口。
- `RuleBasedTestPlanGenerator`：规则测试计划生成。
- `RuleBasedTestExecutor`：模拟执行和日志生成。
- `RuleBasedResultAnalyzer`：模拟漏洞结果生成。
- `MarkdownReportGenerator`：Markdown 报告生成。

## 4. 任务执行流程

1. 用户点击执行测试。
2. 后端清理旧结果、旧日志和旧报告。
3. 状态更新为 `PLANNING`，写入计划生成日志。
4. Agent 生成测试计划。
5. 状态更新为 `RUNNING`，写入执行日志。
6. Agent 模拟执行并生成过程日志。
7. 状态更新为 `ANALYZING`，写入分析日志。
8. Agent 生成漏洞结果。
9. Agent 生成 Markdown 报告。
10. 状态更新为 `COMPLETED`，写入完成日志。

## 注册、用户管理与权限校验详细设计

### 注册流程

用户在注册页填写用户名、密码、确认密码、昵称、邮箱和手机号。前端完成基础校验后调用 `POST /api/auth/register`。后端校验用户名唯一、密码长度和邮箱格式，将密码进行 SHA-256 哈希后保存，默认角色为 USER，状态为 ENABLED。

### 用户管理流程

管理员访问 `/admin/users`，通过 `GET /api/admin/users/page` 分页查询用户。新增、编辑、启用禁用、重置密码和删除分别调用管理员专属接口。后端统一校验当前角色是否为 ADMIN，普通用户访问返回 403。

### 资源权限校验流程

测试任务表通过 `create_user_id` 记录创建人。普通用户查询任务时自动附加创建人过滤条件；查看、编辑、删除、执行任务时校验任务创建人是否为当前用户。漏洞、日志和报告通过 task_id 关联任务，再根据任务创建人判断权限。

## 创新模块详细设计

### Agent 决策链设计

任务执行时，系统清理旧决策链并生成 6 条新记录，阶段包括 `TASK_UNDERSTANDING`、`PLAN_GENERATION`、`TOOL_SELECTION`、`TEST_EXECUTION`、`RESULT_ANALYSIS`、`REPORT_GENERATION`。每条记录包含决策标题、原因、输入摘要、输出摘要、工具名称和置信度。

### 风险评分模型

`RiskScoreService` 根据漏洞名称、漏洞类型和风险等级生成评分。未授权访问约 9.1 分，水平越权约 8.6 分，弱口令约 6.5 分，敏感信息泄露约 5.8 分，参数篡改约 6.2 分。该模型用于课程演示，不等同于完整 CVSS。

### 整改闭环流程

漏洞详情页点击“生成整改工单”，后端根据漏洞结果创建工单。工单可修改状态，模拟复测时若关联漏洞为 FIXED，则工单关闭，否则进入 RETESTING。

### 审计日志流程

认证、任务、漏洞、报告和工单等关键操作调用 `AuditLogService` 写入审计日志。管理员通过 `/api/admin/audit-logs/page` 分页查看。

## 趣味靶场详细设计

### 题目与答案校验

题目保存在 `challenge` 表中，答案不明文存储，只保存 `answer_hash`。用户提交答案时，后端对提交内容进行 SHA-256 哈希后与数据库哈希比对。普通题目接口返回 `ChallengeDTO`，不会下发答案哈希。

### 积分与成就

用户首次答对某题时获得该题积分，重复答对不重复计分。系统根据提交记录刷新 `user_score`，并按规则解锁成就：初次破冰、安全新手、靶场达人、风险猎手和 Agent 观察员。

### 答对动画

前端在 `ChallengeDetail.vue` 中使用 CSS 实现彩带、徽章弹出和成功弹窗。答错时输入区触发轻微抖动，并提示用户查看提示后继续尝试。

### 管理员题目管理

管理员通过 `/api/admin/challenges/**` 管理题目。普通用户访问管理员接口返回 403。管理员可查看题目通过率和提交记录。
