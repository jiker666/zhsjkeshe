# 数据库设计说明书

## 1. 数据库名称

`security_agent_platform`

## 2. 表结构说明

### 2.1 user 用户表

| 字段 | 说明 |
| --- | --- |
| id | 用户 ID |
| username | 用户名 |
| password | 密码 SHA-256 哈希 |
| nickname | 昵称 |
| email / phone | 邮箱和手机号 |
| role | 用户角色 |
| status | 用户状态 |
| create_time | 创建时间 |

### 2.2 test_task 测试任务表

| 字段 | 说明 |
| --- | --- |
| id | 任务 ID |
| task_name | 任务名称 |
| target_url | 目标地址 |
| test_type | 测试类型 |
| description | 任务描述 |
| plan | Agent 生成的测试计划 |
| test_depth | 测试深度 |
| generate_report | 是否生成报告 |
| status | 任务状态 |
| created_by | 创建用户 |
| created_at / updated_at | 创建和更新时间 |

### 2.3 test_result 测试结果表

| 字段 | 说明 |
| --- | --- |
| id | 漏洞结果 ID |
| task_id | 所属任务 |
| vulnerability_name | 漏洞名称 |
| vulnerability_type | 漏洞类型 |
| risk_level | 风险等级 |
| url | 影响地址 |
| description | 漏洞描述 |
| reproduce_steps | 复现步骤 |
| request_example | 请求示例 |
| response_example | 响应示例 |
| suggestion | 修复建议 |
| status | 处理状态 |
| created_at | 创建时间 |

### 2.4 test_log 测试日志表

字段包括日志 ID、任务 ID、日志级别、日志内容和创建时间。日志级别包括 INFO、WARN、ERROR、SUCCESS。

### 2.5 test_report 测试报告表

字段包括报告 ID、任务 ID、标题、内容、格式和生成时间。

## 3. 表关系说明

- `test_task.created_by` 对应 `user.id`。
- `test_result.task_id` 对应 `test_task.id`。
- `test_log.task_id` 对应 `test_task.id`。
- `test_report.task_id` 对应 `test_task.id`。

## 4. 初始化数据说明

`sql/init.sql` 包含 1 个管理员用户、5 个测试任务、10 条漏洞结果、20 条测试日志和 3 份报告，方便答辩时直接展示完整系统效果。

## 双端权限相关表结构更新

### user 表更新

user 表新增或调整字段：nickname、email、phone、role、status、update_time。role 取值为 ADMIN 或 USER，status 取值为 ENABLED 或 DISABLED。password 字段保存 SHA-256 哈希值。

### test_task 表更新

测试任务表使用 `create_user_id` 和 `create_username` 记录任务创建人。ADMIN 可查询全部任务，USER 只能查询 create_user_id 等于当前用户 ID 的任务。

### 表关系说明

`test_result`、`test_log`、`test_report` 均通过 task_id 关联 test_task。系统判断漏洞、日志、报告权限时，先查询关联任务，再判断任务创建人是否为当前用户。

### 初始化数据说明

初始化脚本提供 `admin/123456` 管理员账号和 `user/123456` 普通用户账号。演示任务共 5 条，其中 3 条归属 admin，2 条归属 user，便于答辩演示不同角色的数据隔离效果。

## 创新加分项表结构

### agent_decision

保存 Agent 决策链，字段包括 task_id、step_no、stage、decision_title、decision_reason、input_summary、output_summary、tool_name、confidence、create_time。

### remediation_ticket

保存整改工单，字段包括 result_id、task_id、title、risk_level、status、assignee_id、assignee_name、fix_suggestion、retest_result、create_time、update_time。

### security_knowledge

保存安全知识库，字段包括 vuln_type、title、risk_level、description、cause、impact、fix_advice、reference、create_time。

### audit_log

保存平台操作审计，字段包括 user_id、username、role、action、module、detail、ip、result、create_time。

### test_result 新增字段

漏洞结果表增加 risk_score、impact_score、exploitability_score、difficulty_score、risk_vector，用于风险量化展示和报告输出。

## 趣味靶场表结构

### challenge

题目表。字段包括 id、title、category、difficulty、score、description、target_url、request_method、request_example、hint、answer_hash、explanation、knowledge_id、status、create_user_id、create_time、update_time。answer_hash 使用 SHA-256 保存答案哈希。

### challenge_submit

提交记录表。字段包括 challenge_id、user_id、username、submit_answer、correct、score_got、used_hint、submit_time。用于统计通过率、积分和最近挑战记录。

### user_achievement

用户成就表。字段包括 user_id、achievement_code、achievement_name、achievement_desc、icon、unlock_time。对 user_id 和 achievement_code 建唯一约束，避免重复解锁。

### user_score

用户积分表。字段包括 user_id、username、total_score、solved_count、submit_count、correct_rate、update_time。可由提交记录刷新计算。
