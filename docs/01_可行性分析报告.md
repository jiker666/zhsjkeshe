# 可行性分析报告

## 1. 项目背景

Web 系统安全测试过程通常包含测试任务登记、测试计划编排、执行过程记录、漏洞结果整理和报告输出。传统人工方式过程分散，不利于复盘和展示。本项目设计并实现“基于 AI Agent 的自主安全测试可视化平台”，通过规则模拟 Agent 展示自主安全测试的完整业务闭环。

## 2. 技术可行性

系统采用 Java 17、Spring Boot、MyBatis Plus 和 MySQL 实现后端业务，技术成熟、资料丰富，适合课程设计。前端采用 Vue 3、Element Plus、ECharts 和 Axios，实现安全运营风格页面、分页表格、状态流转、图表展示和报告预览。Agent 第一版采用规则模板模拟，不依赖真实大模型或扫描器，降低开发风险，同时保留后续扩展接口。

## 3. 经济可行性

项目全部采用开源技术栈，本地运行只需要 JDK、Maven、Node.js 和 MySQL，无需购买云服务、商业组件或大模型 API，开发成本和演示成本较低。

## 4. 操作可行性

系统使用浏览器访问，演示流程清晰：登录、查看仪表盘、新建任务、执行测试、查看漏洞、查看日志、预览报告。前端提供示例任务一键填充和初始化演示数据，降低答辩现场操作难度。

## 5. 安全合规可行性

系统默认面向课程演示环境、本地靶场或已授权目标，不进行破坏性扫描，不执行高频请求，不绕过授权边界。第一版 Agent 只生成模拟日志、模拟漏洞和修复建议，符合课程设计的安全合规要求。

## 6. 结论

项目范围适中、技术路线清晰、成本可控、演示价值明显，能够体现软件工程综合实践中的可行性分析、需求分析、系统设计、实现、测试和文档交付过程，具备较高课程设计可行性。
